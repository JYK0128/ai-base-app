# 02. 테넌트 서비스 스펙 (요약)

## 1. 개요

* 파트너사(Tenant)의 라이프사이클 관리 및 시스템 전반의 핵심 정책 집행

## 2. 주요 기능

* **온보딩/프로비저닝**: 파트너 가입 승인, 전용 DB 스키마 및 K8s 리소스(Namespace 등) 자동 생성
* **구독 및 쿼터 관리**: 티어별 기능 제한 및 리소스 사용량 집계, 빌링 서비스 연동
* **조직 및 멤버 관리**: 파트너사 내부 구성원 초대(초대 토큰 기반) 및 권한 그룹 관리

## 3. 기술적 요구사항

* **원자성 보장**: Outbox 패턴을 활용한 인프라 할당과 비즈니스 데이터 저장의 원자적 처리
* **인프라 자동화**: Helm 또는 Terraform 연동을 통한 테넌트별 독립 리소스 배포 자동화
* **격리 설계**: 모든 테넌트 데이터에 대한 Row-Level Security(RLS) 적용 지원

## 4. 핵심 API

* `setup-tenant`: 신규 테넌트 환경 인프라 구성 요청
* `usage-metrics`: 테넌트별 실시간 리소스 이용률 및 할당량 조회
* `invite-member`: 새로운 운영진 초대 및 초대 프로세스 관리
