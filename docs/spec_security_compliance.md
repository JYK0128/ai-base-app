# 보안 및 컴플라이언스 사양서 (Startup-Friendly MVP)

## 1. 개요

스타트업의 한정된 리소스를 고려하여, 적은 공수로 최대의 보안 효과(ISMS-P 수준 준수)를 낼 수 있는 실무 중심의 보안 사양을 정의합니다.

## 2. 핵심 데이터 보호 (Data Protection)

* **표준 개인정보 암호화**: 성명, 전화번호 등 민감 정보는 국가 표준 알고리즘(AES-256)을 사용하여 DB 저장 시 암호화합니다.
* **비밀번호 단방향 해싱**: 사용자 비밀번호는 **Bcrypt**를 사용하여 복호화가 불가능하게 처리합니다.
* **통신 암호화 (SSL/TLS)**: 모든 외부 트래픽은 HTTPS(TLS 1.2 이상)를 강제합니다.

## 3. 접근 권한 통제 (Access Control)

* **표준 MFA 도입**: 관리자 페이지 로그인 시 **이메일 또는 문자 인증**을 통한 2차 인증을 필수로 설정합니다.
* **네트워크 격리 (VPC)**: 데이터베이스와 어플리케이션 서버를 외부에서 직접 접근할 수 없는 내부망(Private Subnet)에 배치합니다.
* **RBAC**: 기본적인 역할(Admin, Staff)에 따른 메뉴 접근 권한을 분리합니다.

## 4. 실전형 개발 보안 (Dev Security)

* **자동 취약점 스캔**: GitHub Actions 등 CI/CD 과정에서 알려진 보안 취약점(Snyk 등)을 무료/저비용 툴로 자동으로 점검합니다.
* **시크릿 스캐닝**: 코드 내에 API 키나 자격 증명이 실수로 포함되어 커밋되는 것을 방지하기 위해 자동 스캔을 수행합니다.

## 5. 개인정보 및 규정 준수 (Privacy)

* **표준 마스킹**: 화면 로딩 시 전화번호 끝자리 등을 마스킹(`***`) 처리하여 불필요한 노출을 방지합니다.
* **로그 보존 및 파기**: 사용 로그는 최소 1년 보관하며, 서비스 탈퇴 시 관련 정보를 지체 없이 파기하는 표준 절차를 준수합니다.

## 6. 감사 로그 (Audit Logs)

* **행위 기록**: 관리자의 주요 데이터 조회(엑셀 다운로드 등) 및 변경 이력을 로그로 기록합니다.
* **로그 백업**: 로그 데이터는 운영 서버와 분리된 안정적인 스토리지에 정기적으로 백업합니다.

## 7. 인프라 보안 (Infra Security)

* **표준 방화벽 (SG)**: 필수 포트(80, 443) 외의 모든 인바운드 트래픽을 차단합니다.
* **정기 백업**: 매일 1회 DB 스냅샷 백업을 수행하여 장애 발생 시 신속한 복구 체계를 갖춥니다.

## 8. 시크릿 및 키 관리 체계 (Secret Management)

* **환경변수 기반 관리**: 모든 API 키, DB 비밀번호는 소스 코드와 분리하여 서버 환경변수(Env) 또는 K8s Secrets를 통해 관리합니다.
* **표준 시크릿 매니저**: 민감한 자격 증명은 클라우드 네이티브 시크릿 매니저 또는 전용 보안 저장소를 통해 중앙 집중 관리합니다.
* **최소 권한 할당**: 각 서비스 및 개발자는 업무에 필요한 최소한의 시크릿에만 접근할 수 있도록 권한을 분리합니다.

## 9. 대응 프로세스 (Response)

* **비상 연락망 구축**: 보안 사고 발생 시 즉각 대응할 수 있는 내부 비상 연락망 및 대응 순서를 정의합니다.
* **재해 복구 방식**: 표준 백업본을 활용한 서비스 복구 절차를 수립합니다.
