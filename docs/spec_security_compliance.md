# 보안 및 컴플라이언스 사양서 (Lean Startup MVP)

## 1. 개요

스타트업의 한정된 리소스를 고려하여, 적은 관리 공수로 필수 보안 요건을 충족하는 **'생존형 보안 아키텍처'**를 정의합니다.

## 2. 핵심 데이터 보호 (Data Protection)

* **필수 개인정보 암호화**: AES-256 알고리즘을 사용하여 DB 내 민감 정보를 암호화합니다.
* **비밀번호 단방향 해싱**: **Bcrypt**를 사용하여 복호화가 불가능하게 처리합니다.
* **데이터 리전 고정**: 모든 데이터와 백업본은 **대한민국 내 리전**에 보관함을 원칙으로 합니다.

## 3. 실용적인 접근 통제 (Access Control)

* **표준 TOTP 2차 인증**: 관리자 페이지 로그인 시 **OTP 앱(Google Authenticator 등)**을 통한 2차 인증을 적용합니다. (발송 비용 0원, 높은 보안성)
* **네트워크 격리 (VPC)**: 외부 노출이 필요 없는 DB 등은 내부망에 배치하여 물리적 공격 지점을 줄입니다.
* **최소 권한 할당**: 개발자에겐 꼭 필요한 시크릿 정보만 공유하여 보안 사고 영향을 최소화합니다.

## 4. 빌트인 개발 보안 (Built-in Security)

* **플랫폼 내장 보안 스캔**: GitHub Dependabot 등 **플랫폼에서 제공하는 무료 보안 스캔** 기능을 활성화하여 별도 관리 공수를 없앱니다.
* **환경변수 관리**: API 키 등은 소스 코드에 절대 포함하지 않고 환경변수로만 관리합니다.

## 5. 개인정보 및 규정 준수 (Privacy)

* **선택적 마스킹**: 전화번호 등 주요 정보만 화면에서 마스킹(`***`) 처리합니다.
* **유연한 로그 보존**: 핵심 행위 로그(결제 등)는 1년 보존하되, 일반 로그는 **90일 후 자동 삭제**하여 비용을 절감합니다.

## 6. 인프라 보안 및 백업 (Infra & Backup)

* **기본 방화벽**: 필수 포트 외 전면 차단 정책을 적용합니다.
* **자동 백업 모니터링**: 매일 자동 백업을 수행하며, **백업 실패 시에만 슬랙(Slack)으로 알림**을 받아 대응합니다.

## 7. 대응 체제 (Response)

* **슬랙 비상 채널**: 보안 사고 탐지 시 즉각 소통할 수 있는 전용 슬랙 알림 채널을 운영합니다.
* **가이드라인 README**: 복잡한 SOP 대신, 장애 발생 시 조치 요령을 담은 심플한 **README 문서**를 유지합니다.
