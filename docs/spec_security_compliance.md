# 보안 및 컴플라이언스 사양서 (Lean Startup MVP)

## 1. 개요

스타트업의 한정된 리소스를 고려하여, 적은 관리 공수로 필수 보안 요건을 충족하는 **'생존형 보안 아키텍처'**를 정의합니다.

## 2. 핵심 데이터 보호 (Data Protection)

* **필수 개인정보 암호화**: AES-256 알고리즘을 사용하여 DB 내 민감 정보를 암호화합니다.
* **비밀번호 단방향 해싱**: **Bcrypt**를 사용하여 비밀번호를 안전하게 저장합니다.

## 3. 실용적인 접근 통제 (Access Control)

* **표준 TOTP 2차 인증**: 관리자 페이지 로그인 시 **OTP 앱(Google Authenticator 등)**을 통한 2차 인증을 적용합니다. (발송 비용 0원, 높은 보안성)
* **네트워크 격리 (VPC)**: 외부 노출이 필요 없는 DB 등은 내부망에 배치하여 물리적 공격 지점을 줄입니다.
* **최소 권한 할당**: 개발자에겐 업무에 꼭 필요한 시크릿 정보만 공유하여 보안 사고 영향을 최소화합니다.

## 4. 고효율 보안 필살기 (High ROI Security)

* **API 속도 제한 (Rate Limiting)**: 무차별 로그인 시도를 차단 위해 동일 IP/계정에 대한 요청 횟수를 제한합니다.
* **보안 헤더 일괄 적용 (Helmet)**: XSS 및 클릭재킹 등 웹 공격 방지를 위해 표준 보안 헤더를 서버 전반에 적용합니다.
* **심플 활동 원장 (Activity Log)**: 데이터 삭제/수정 등 주요 행위에 대해 '누가, 언제' 작업을 수행했는지 최소한의 추적 로그를 남깁니다.

## 5. 빌트인 개발 보안 (Built-in Security)

* **플랫폼 내장 보안 스캔**: GitHub Dependabot 등 플랫폼 제공 보안 스캔 기능을 활성화하여 별도 관리 공수 없이 취약점을 감시합니다.
* **환경변수 관리**: API 키 등은 소스 코드에 절대 포함하지 않고 서버 환경변수로만 관리합니다.

## 6. 개인정보 및 규정 준수 (Privacy)

* **선택적 마스킹**: 전화번호 등 주요 정보만 화면에서 마스킹(`***`) 처리하여 불필요한 노출을 방지합니다.
* **유연한 로그 보존**: 핵심 행위 로그는 1년 보존하되, 일반 로그는 **90일 후 자동 삭제**하여 스토리지 비용을 절감합니다.

## 7. 인프라 보안 및 백업 (Infra & Backup)

* **기본 방화벽**: 필수 서비스 포트 외 모든 인바운드 트래픽을 차단하는 보안 그룹(SG) 정책을 적용합니다.
* **자동 백업 모니터링**: 벤더 제공 자동 백업을 수행하며, **백업 실패 시에만 슬랙(Slack)으로 알림**을 받아 대응합니다.

## 8. 가용성 및 네트워크 보안 (Network & Availability)

* **표준 DDoS 방어**: 클라우드 네이티브 DDoS 보호 기능을 활성화하여 비정상 트래픽 공격으로부터 서비스를 보호합니다.
* **CORS 보안 정책**: 허용되지 않은 도메인에서의 API 호출을 차단하여 CSRF 등 브라우저 기반攻擊을 방어합니다.
* **데이터 소프트 삭제**: 주요 데이터 삭제 시 물리적 삭제 대신 **삭제 플래그(Soft-delete)**를 사용하여 실수 방지 및 추적성을 확보합니다.

## 9. 보안 디테일 및 무결성 (Security Details & Integrity)

* **단기 수명 토큰 정책**: 액세스 토큰의 유효 기간을 30분 이내로 짧게 유지하고 리프레시 토큰을 통해 갱신하는 보안 표준을 준수합니다.
* **ORM 기반 인젝션 방어**: 로우 쿼리 대신 ORM(Mikro-ORM)의 파라미터 바인딩 기능을 활용하여 SQL Injection을 기술적으로 차단합니다.
* **패키지 무결성 잠금**: `pnpm-lock.yaml`을 통해 사용 라이브러리 버전을 고정하여 소프트웨어 공급망의 무결성을 보장합니다.

## 10. 보안 운영 완성도 (Operational Excellence)

* **보안 쿠키 정책**: 브라우저 내 토큰 탈취 방지를 위해 `HttpOnly`, `Secure`, `SameSite=Strict` 옵션을 강제합니다.
* **로그 개인정보 마스킹**: 로그 기록 시 비밀번호, 인증 키 등 민감한 패턴을 자동으로 탐지하여 마스킹 처리함으로써 로그를 통한 유출을 방지합니다.
* **API 버전 관리**: API 엔드포인트에 버전(예: `/v1/`)을 명시하여 보안 업데이트 및 구버전 폐기를 용이하게 관리합니다.

## 11. 시스템 무결성 및 가용성 디테일 (System Integrity)

* **요청 크기 제한 (Size Limiting)**: 대용량 페이로드 전송을 통한 자원 고갈 공격을 방지하기 위해 각 엔드포인트별 요청 본문 크기를 엄격히 제한합니다.
* **헬스체크 및 우아한 종료**: 컨테이너의 상태를 실시간 감시(Health Check)하고, 종료 시 진행 중인 작업을 안전하게 마무리(Graceful Shutdown)하여 데이터 무결성을 유지합니다.

## 12. 마이크로 보안 및 노출 방지 (Exposure Prevention)

* **프로덕션 에러 상세 노출 금지**: 운영 환경에서 발생한 오류 발생 시 최소한의 서버 정보만 사용자에게 노출하여 시스템 내부 구조 파악을 방지합니다.
* **검색 엔진 노출 통제(robots.txt)**: `/admin`, `/api` 등 내부 주요 주소에 대해 검색 엔진 로봇의 인덱싱을 차단하여 불필요한 정보 노출을 방어합니다.
* **도메인 CAA 레코드 설정**: 도메인 레벨에서 인증서 발급 기관을 명시하여 권한 없는 타 기관에서의 허위 인증서 발급 시도를 차단합니다.

## 13. 보안 습관 및 전송 보안 (Secure Habits)

* **서버 익명성 유지**: 응답 헤더에서 `X-Powered-By` 등의 서버 기술 정보를 제거하여 불필요한 정보 제공을 방지합니다.
* **민감 데이터 전송 원칙**: 토큰, 비밀번호 등 민감 정보는 어떠한 경우에도 URL Query String에 노출하지 않으며 항상 Body를 통해 전송합니다.
* **파일 업로드 객체 검증**: 파일 업로드 시 확장자뿐만 아니라 실제 MIME Type을 검증하여 허용되지 않은 형식의 파일(스크립트 등) 유입을 차단합니다.

## 14. 운영 및 배포 보안 고도화 (Ops & Deployment)

* **배포 환경 승인 정책**: 운영 환경(Production) 배포 시 지정된 관리자의 명시적 승인을 거치도록 GitHub 배포 프로세스를 강화합니다.
* **관리자 접속 대역 제한**: 관리자 전용 서비스에 대해 사무실 IP 등 신뢰할 수 있는 특정 대역에서의 접근만 허용하도록 화이트리스트 정책을 적용합니다.
* **관리 지원 행위 로깅**: 운영자 고객 대리 접속이나 중요 설정 변경 시, 별도의 사유 입력과 함께 모든 행위를 정밀하게 기록합니다.

## 15. 사용자 환경 및 데이터 수명 관리 (Lifecycle Security)

* **휴면 계정 자동 관리**: 장기 미접속 계정에 대해 접근 권한을 자동으로 정지하거나 데이터를 분리 보관하여 개인정보 유출 위험을 줄입니다.

## 16. 사용자 신뢰 및 토큰 보안 (Trust & Token)

* **리프레시 토큰 로테이션**: 새로운 액세스 토큰 발급 시 리프레시 토큰도 함께 갱신 및 교체하여 토큰 재사용을 통한 침입을 방지합니다.
* **재설정 링크 단회성 보장**: 비밀번호 재설정 등 보안 민감 링크는 1회 사용 즉시 무효화하여 이메일 탈취 등으로 인한 2차 피해를 차단합니다.
* **사용자 보안 이벤트 알림**: 로그인 기기 변경, 비밀번호 수정 등 주요 계정 이벤트 발생 시 즉각 사용자에게 알림을 발송하여 계정 자가 감시를 지원합니다.

## 17. 보안 거버넌스 및 선순환 체계 (Governance & Evolution)

* **직원 보안 인식 교육**: 전사적 보안 수칙 습관화를 위해 정기적인 사내 보안 가이드 배포 및 인적 오류 방지 교육을 실시합니다.
* **주기적 자가 보안 점검**: 분기별 보안 설정 검토 및 종속성 취약점(pnpm audit) 점검을 통해 보안 수준을 최신 상태로 유지합니다.
* **사고 사후 분석 및 개선**: 경미한 보안 이슈 발생 시에도 상세 원인 분석(Post-mortem)을 수행하고, 도출된 개선 사항을 개발 프로세스에 환류합니다.

## 18. 비즈니스 신뢰성 보안 (Business Trust)

* **멱등성 결제 처리 (Idempotency)**: 네트워크 오류 등으로 인한 중복 결제를 방지하기 위해 고유 요청 키를 기반으로 단 1건의 트랜잭션만 처리됨을 보장합니다.

## 19. 실무 지향 보안 강화 (Practical Hardening)

* **지역 기반 접근 제한 (Geo-blocking)**: 불필요한 국외 IP 대역의 접근을 차단하여 글로벌 자동화 봇에 의한 무차별 공격 노출을 최소화합니다.
* **개발 데이터 비식별화**: 개발 및 테스트 환경 구축 시 실제 운영 데이터를 가상 데이터로 치환하여 개발 과정에서의 정보 유출 위험을 원천 차단합니다.
* **엄격한 TLS 1.2+ 준수 정책**: 구형 보안 프로토콜의 취약성을 방어하기 위해 최신 전송 보안 표준(TLS 1.2 이상)만 허용하도록 서버 통신을 강제합니다.

## 20. 마이크로 보안 실무 (Micro-Security)

* **일회용 가공 이메일 차단**: 봇 계정 유입 및 스패머 방지를 위해 임시 이메일 도메인을 통한 가입을 필터링합니다.
* **최근 로그인 정보 노출**: 사용자 프로필 페이지에 마지막 접속 시간, 위치 정보를 제공하여 계정 도용 여부를 사용자 스스로 감시하게 유도합니다.
* **민감 작업 재인증 정책**: 비밀번호 변경 등 보안 민감 액션 시 세션 상태와 무관하게 현재 비밀번호를 다시 확인합니다.

## 21. 실무 보안 디테일 패키지 (Security Details)

* **무작위 고유 식별자(UUID/ULID)**: API 엔드포인트 및 DB PK 노출 시 순차적 ID 대신 UUID를 사용하여 자원 열거 공격을 원천 차단합니다.
* **HSTS 웹 보안 강제 적용**: 브라우저 레벨에서 강제 HTTPS 통신 정책(HSTS)을 활성화하여 평문 통신 구간이 발생하지 않도록 보장합니다.
* **세션 고정 공격 방어**: 로그인 성공 직후 세션 ID를 강제 재생성하여 가로채기나 고정 공격을 기술적으로 무력화합니다.
* **오픈 리다이렉션 방어**: 외부 URL로의 리다이렉션 시 신뢰할 수 있는 도메인 리스트를 검증하여 피싱 사이트 유도를 차단합니다.
* **가입 확인 필수 이메일**: 회원 가입 및 주요 정보 변경 시 이메일 소유권 확인 절차를 필수화하여 유효 사용자 기반을 유지합니다.
* **취약점 제보 채널(security.txt)**: 외부의 선의의 보안 제보자와 소통할 수 있는 표준 규격의 `/security.txt`를 운영합니다.

## 22. 지능형 방어 및 사용자 보호 (Intelligent Defense)

* **미사용 세션 자동 로그아웃**: 자리를 비운 사이의 불법 접속을 방어하기 위해 일정 시간 동안 활동이 없는 세션을 자동으로 파기하는 타임아웃 정책을 적용합니다.
* **신규 로그인 환경 자가 감시**: 평소와 다른 기기, 지역 및 새로운 IP 대역에서의 로그인 발생 시 즉각 사용자에게 알림을 발송하여 계정 자가 보안 감시를 지원합니다.

## 23. 운영 정책 및 심리적 보안 (Policy & Trust)

* **계정 잠금 정책 (Account Lockout)**: 무차별 대입 공격 방지를 위해 로그인 연속 5회 실패 시 해당 계정을 30분간 임시 잠금 처리합니다.
* **비밀번호 위생 정책 (Sanitization)**: 유추하기 쉬운 비밀번호나 과거 사용 비밀번호의 재사용을 금지하는 정교한 강도 검증 로직을 적용합니다.
* **보안 약속 공개 가공 (Public Transparency)**: 비즈니스 신용 확보를 위해 "우리는 고객 데이터를 어떻게 보호하는가"에 대한 보안 백서를 공개하여 사용자 및 B2B 협력사의 신뢰를 구축합니다.

## 24. 정밀 운영 및 로봇 방어 (Ops & Bot Defense)

* **로그아웃 시 브라우저 완전 세정(Clear-Site-Data)**: 로그아웃 시 브라우저 전용 헤더를 통해 쿠키, 스토리지, 캐시를 일괄 강제 파기하여 잔류 정보 유출을 방지합니다.
* **로봇 유인 함정(Robots.txt Honeypot)**: `robots.txt`에 명시된 비공개 경로에 접근하는 비정상 로봇/스캐너 IP를 자동 탐지하여 접근 차단 리스트에 등록합니다.
* **동시 세션 개수 제한**: 한 계정당 허용되는 동시 활성 세션 수를 제한하여 계정 공유 및 도용 범위를 축소합니다.
* **비정상 User-Agent 필터링**: 알려진 공격 도구 및 비정상 브라우저 정보를 가진 요청을 수신 거부하여 자동화된 취약점 스캔을 억제합니다.
* **세션-IP 바인딩(Session Pinning)**: 활성 세션의 IP 대역이 비정상 변경될 경우 세션을 강제 종료하여 가로채기 방식의 하이재킹을 무력화합니다.

## 25. 코드 및 환경 하드닝 (Code & Env Hardening)

* **코드 주석 및 환경변수 부비트랩(Honeypot Decoy)**: 프론트엔드 코드나 설정 내 가짜 API 키/주소를 배치하고, 이에 대한 접근 발생 시 즉각 해당 IP를 영구 차단 리스트에 등록합니다.
* **타이밍 공격 및 전수 조사 방어(Timing Defense)**: 로그인 실패 시 의도적으로 일정한 지연 시간(0.3~0.5초)을 두어 자동화 도구의 공격 속도를 늦추고 내부 로직 정보를 은닉합니다.
* **비정상 활동 수치 자동 차단(Scraping Prevention)**: 평소 유저 행동 범위를 과도하게 벗어난 대량의 데이터 조회 패턴 발생 시 계정을 즉각 임시 정지하고 정밀 조사를 수행합니다.
* **VPN 및 공개 프록시 우회 공격 차단**: 신원 은닉을 위한 클라우드 서버 대역(AWS, GCP 등) 및 공개 대역 VPN IP로부터의 접속을 제한하거나 엄격히 검증합니다.
* **보안 지원 용도의 한계 시간 권한 부여**: 운영직군의 지원용 대리 접속 권한 등은 사용 즉시 타이머가 작동하여 최대 1시간 후 자동 회수되도록 강제합니다.

## 26. 포렌식 및 법적 증거 보존 (Forensics)

* **로그 무결성 보존 정책(Write-Once)**: 사후 포렌식 및 법적 증거 유효성 확보를 위해 한 번 기록된 보안 로그는 수정 및 삭제가 불가능한 환경에 별도 보관합니다.

## 27. 대응 체제 (Response)

* **슬랙 비상 채널**: 보안 사고 탐지 시 즉각 소통할 수 있는 전용 슬랙 알림 채널을 운영합니다.
