# Core Service 상세 사양서 (MVP Edition)

## 1. 개요

Core Service는 멀티테넌트 환경의 근간이 되는 조직 관리 및 기본 권한 체계를 담당합니다.

## 2. 상세 도메인 모델

### 2.1. Organization (조직)

* **테넌트 격리**: 모든 데이터는 `organizationId`를 기준으로 격리됩니다.
* **상태 관리**: ACTIVE, SUSPENDED, PENDING 등 표준 상태를 가집니다.

### 2.2. 권한 관리 (RBAC)

* **표준 역할(System Role)**: 기본적으로 `Owner`, `Admin`, `Manager` 역할을 제공하며, 이는 시스템에 의해 보호되는 역할입니다.
* **커스텀 역할(Custom Role)**: 조직별로 고유한 역할을 무제한 생성할 수 있습니다.
* **다중 역할 부여**: 한 명의 매니저에게 **여러 개의 역할(표준 + 커스텀)**을 동시에 부여하여 복합적인 권한 체계를 구성할 수 있습니다.
* **권한 식별**: 소속 조직의 유형(PLATFORM/PARTNER)과 할당된 역할들의 권한 셋(Set)을 합산하여 최종 접근 권한이 결정됩니다.
* **초대 시스템**: 이메일을 통해 신규 관리자를 초대하는 기능을 지원합니다.

## 3. 핵심 API 엔드포인트

| Method | Path | Description |
| :--- | :--- | :--- |
| `POST` | `/organizations` | 신규 조직 가입 및 초기화 |
| `GET` | `/organizations/:id/config` | 로고 및 기본 설정 조회 |
| `POST` | `/staff/invite` | 매니저 초대장 이메일 발송 |

## 4. 보안 및 규정 준수

* **타임존 및 언어**: 각 조직별로 고유한 타임존과 기본 언어를 설정합니다.
* **데이터 마스킹**: 개인정보보호를 위해 민감한 데이터는 마스킹 처리하여 저장합니다.

## 5. 데이터 관리 및 소통

* **표준 데이터 내보내기**: 파트너사가 자사 데이터를 CSV 형식으로 다운로드할 수 있는 기능을 제공합니다.
* **공지사항 및 알림**: 플랫폼의 주요 변경 사항을 포털에 공지하고 이메일로 발송합니다.
