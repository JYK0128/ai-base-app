# 보안 및 컴플라이언스 사양서 (요약)

## 1. 개요

* ISMS-P 인증 기준 및 제로 트러스트(Zero Trust) 모델 기반의 전사적 통합 보안 아키텍처 정의

## 2. 데이터 및 전송 보호

* **암호화**: AES-256(민감정보), Bcrypt(비밀번호), TLS 1.3(통신) 적용
* **마스킹**: 화면 및 로그 내 개인정보 패턴 자동 식별 및 마스킹 처리

## 3. 접근 및 권한 제어

* **인증**: 관리자 TOTP 2FA 필수, 단기 수명 JWT 및 Refresh Token 로테이션
* **인프라**: VPC 기반 네트워크 격리, SG(보안 그룹) 최소 포트 개방
* **권한**: 업무별 최소 권한 할당(PoLP) 및 관리자 접속 IP 대역 제한

## 4. 실전 방어 및 안정성

* **웹 보안**: Helmet(보안 헤더), CORS 정책, SQL Injection 방어(ORM 파라미터 바인딩)
* **공격 차단**: IP/계정별 Rate Limiting, DDoS 방어 서비스, Geo-blocking(해외 유입 차단)
* **무결성**: `pnpm-lock` 기반 라이브러리 고정, 파일 MIME Type 검증

## 5. 보안 운영 및 관측성

* **감사**: 모든 주요 행위(CUD)의 불변 감사 로그(Audit Trail) 1년 보존
* **스캔**: GitHub Dependabot 취약점 자동 감시 및 환경변수(Secret) 중앙 관리
* **대응**: 보안 이벤트 발생 시 전용 슬랙(Slack) 채널 실시간 알림 및 백업 감시
