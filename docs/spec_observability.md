# 관측성 및 모니터링 사양서 (MVP Edition)

## 1. 개요

시스템의 가동 상태를 확인하고 장애 발생 시 빠르게 감지하기 위한 표준 기술 명세입니다.

## 2. 응답 상태 모니터링

* **HTTP 5xx 감시**: 전역 인터셉터를 통해 발생하는 모든 서버 에러(500번대)를 집계합니다.
* **성능 지연 감지**: 응답 속도가 3초 이상 소요되는 API 요청을 로깅합니다.

## 3. 구조화된 로깅

모든 백엔드 로그는 표준 포맷으로 출력하여 검색 및 분석이 가능하게 합니다.

* 필수 필드: `timestamp`, `level`, `userId`, `serviceName`, `orgId`, `message`, `clientIp`.

## 4. 로그 무결성 및 보존 (Log Integrity)

보안 사양서 제26항에 의거하여, 마켓플레이스의 신뢰성을 위해 감사 로그의 무결성을 보장합니다.

* **Write-Once 정책**: 침입자나 내부자에 의한 증거 인멸을 방어하기 위해, 한 번 기록된 보안/감사 로그는 지정된 보존 기간 동안 수정이나 삭제가 불가능한 환경(WORM 스토리지 등)에 별도 저장됩니다.
* **포렌식 대응**: 모든 행동 로그는 법적 증거 유효성을 고려하여 오염되지 않은 상태로 1년간 필수 보존됩니다.

## 5. 장애 알림 (Alerting)

* **에러 임계치 알림**: 분당 일정 횟수 이상의 에러 발생 시 개발팀 슬랙(Slack) 채널로 자동 알림을 전송합니다.
* **보안 위협 감지 알림**: Honeypot 접근, Brute-force 공격 감지 시 즉각 비상 슬랙 알림을 가동합니다.
* **인프라 감시**: CPU 및 메모리 사용량이 노드 한계치에 도달할 경우 경고를 발생시킵니다.
