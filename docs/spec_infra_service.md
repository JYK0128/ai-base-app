# Infrastructure Service 상세 사양서 (Cloud-Agnostic MVP)

## 1. 개요

Infra Service는 특정 벤더에 종속되지 않는 표준 프로토콜(SMTP, S3 API)을 사용하여 알림 발송 및 파일 스토리지를 관리합니다.

## 2. 주요 기능 상세

### 2.1. 통합 알림 발송 (Messaging)

* **표준 SMTP 발송**: 가입 환영, 비밀번호 찾기 등 안내 메일을 표준 SMTP 프로토콜을 통해 발송합니다. (특정 벤더 SDK 배제)
* **카카오 알림톡/문자**: 국내 표준 메시징 게이트웨이 API와 연동하며, 발송 실패 시 자동 전환(Fail-back) 로직을 갖춥니다.

### 2.2. 화이트라벨링 발송 커스터마이징

* **발신인 오버라이드**: 파트너 조직별로 자신의 공식 이메일 주소, 발송 번호, 카카오 채널을 등록하여 운영할 수 있습니다.

### 2.3. 범용 오브젝트 스토리지 (Storage)

* **S3 Compatible API**: 표준 S3 API 규격을 준수하는 모든 오브젝트 스토리지(오픈소스 MinIO, 전용 스토리지 등)와 호환됩니다.
* **Presigned URL**: 서버 인터페이스를 통해 파일 업로드/다운로드 권한을 표준 방식으로 제어합니다.

### 2.4. 활동 로그 및 보안

* **Audit Log**: 관리자의 주요 행위와 접속 IP를 표준 로그 포맷으로 기록합니다.
* **Secret Management**: 민감한 자격 증명은 표준 보안 관리 솔루션(Vault 또는 K8s Secrets)을 통해 관리합니다.

## 3. 핵심 API 명세

* `/notifications/send`: 통합 알림 발송 요청
* `/files/presign`: 스토리지 업로드용 권한 주소 발급
* `/audit-logs`: 전사 마이크로서비스 통합 활동 기록 조회 (Write-Once 저장소 기반)

## 4. 기술 구현 원칙

* **Standard Protocols Over Libraries**: 특정 클라우드 라이브러리 대신 표준 네트워크 프로토콜을 사용하여 인프라 이식성을 극대화합니다.
