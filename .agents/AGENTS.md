# Agent Guidelines

공유 환경(Shared Environment) 기반 작업 수칙.

---

## 📋 Interaction Protocols

## 1. **한국어 우선 (Korean First)**

- 모든 답변, 문서, 주석 등 모든 출력물은 한국어 작성을 원칙으로 함.
- [필수] 모든 답변은 서술형을 배제하고, 반드시 명확하고 간결한 **개조식(Bullet points)** 형식을 사용하여 보고함.
- 오류 발생 시 즉시 모든 작업을 중단하고, 추가적인 독자적 조사나 수정 없이 원인 분석 결과와 해결 방안을 사용자에게 즉시 보고한 후 승인을 받아 진행함.

## 2. **터미널 공유 (Terminal Sharing)**

- 제공된 **`ProcessId`** 기반으로 활동 조율 및 실시간 상태 추적.
- 기존 서버 프로세스 임의 종료 절대 금지.
- 중복 터미널 생성을 지양하고 활성 세션 내에서 작업함.

## 3. **브라우저 공유 (Browser Sharing)**

- 제공된 활성 세션(**`PageId`**) 및 공유 페이지를 반드시 활용함.
- 독립 브라우저 실행을 금지하고 사용자 화면과 컨텍스트를 동기화함.
- UI 검증은 항상 공유된 브라우저 페이지에서 수행함.

## 4. **상태 지속성 (State Persistence)**

- 대화 간 **`ProcessId`**, **`PageId`**를 추적하여 작업 컨텍스트를 완벽히 유지함.

## 5. **작업 범위 (Scope of Work)**

- 문서 검토 후, 백엔드(API) 및 프론트엔드(UI) 작업을 통합하여 유기적으로 수행함.
- 엔티티 설계부터 브라우저 검증까지 포함된 완결된 결과물을 전달함.
- 구현 전후 내용에 맞춰 관련 기술 문서(사양서, 아키텍처 등)를 반드시 최신화함.

## 6. **적응형 문제 해결 (3-Strike Rule)**

- 동일 동작 실패 시 3회를 초과하여 반복하지 않음.
- 실패 시 즉시 모든 도구 호출을 중단하고 로그를 분석하여 사용자에게 상황을 보고함. (추가 도구 호출 금지)
- 분석된 대안(스크립트 방식 등)을 사용자에게 보고하여 확인받은 후 작업을 재개함.

## 7. **문서 기반 작업 (Documentation First)**

- 모든 기술적 의사결정 및 구현 시 `.docs/` 디렉토리 내의 사양서, 아키텍처, 개발 표준을 상시 참조함.
- 특히 `05_references` 폴더의 인터페이스/백엔드/프론트엔드 표준을 최우선으로 준수하여 프로젝트의 일관성을 유지함.

## 8. **파일 관리 및 임시 작업 (File Management & Temp Works)**

- 디버깅, 가동 테스트, **설치 도구(바이너리, 스크립트 등)**를 포함하여 작업 중 발생하는 모든 임시 파일은 반드시 루트의 **`.tmp/`** 디렉토리에서 관리함.
- 프로젝트 루트에 직접 임시 파일을 생성하는 것을 금지하며, 작업 종료 후에는 해당 파일을 정리하여 프로젝트 환경을 깨끗하게 유지함.
