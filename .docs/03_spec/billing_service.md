# 06. 빌링 서비스 스펙 (요약)

## 1. 개요

* 파트너사 구독 모델 관리 및 사용량 기반 과금(Metered Billing) 시스템 운영

## 2. 주요 기능

* **구독 관리**: 멤버십 플랜 정의, 상태 전환(활성/만료/연체), 플랜 변경 시 일할 계산 처리
* **결제 프로세스**: 외부 PG 연동을 통한 정기/단건 결제 및 환불(Partial/Full) 기능 제공
* **인보이스 발행**: 실시간 사용량 집계 기반 월간 인보이스 자동 생성 및 전송

## 3. 기술적 요구사항

* **정밀도/무결성**: 소수점 데이터 정밀도 유지 및 중복 결제 방지를 위한 멱등성 키(Idempotency) 적용
* **보안 준수**: PCI-DSS 기준 카드 정보 직접 저장 금지 (외부 PG 토큰 방식)
* **이벤트 연동**: 결제 완료 시 테넌트 권한 업데이트를 위한 시스템 간 이벤트 발행

## 4. 핵심 API

* `plans`: 사용 가능한 구독 모델 정보 제공
* `subscribe/cancel`: 구독 시작 및 해지 프로세스
* `invoices`: 테넌트별 과금 상세 내역 및 증빙 관리
